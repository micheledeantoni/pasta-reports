# ATT Visual Payload Contract

This note documents the current frontend contract for ATT role-page spatial visuals.
It is a frontend contract only: payload generation and backend/export validation live outside this static site.

## ATT Spatial Panels

ATT pages use the shared role-page template and `assets/js/sr-role-runtime.js`, with ATT-specific labels generated by `generate_pages.py`.

| Panel label | Runtime hook | Payload field | What it renders |
|---|---|---|---|
| `Zone di tiro` | `pitchPos` | `shotField` | Aggregated ATT shot-zone profile from a 12x8 grid. |
| `Impronta offensiva` | `pitchCarry` | `ip` | General attacking/offensive density. |
| `Distribuzione passaggi` | `pitchPass` | `pass` | Pass distribution and direction grid. |
| `Direzione conduzioni` | `pitchProg` | `carry` | Carry direction grid. |
| `Zone placement tiri` | `pitchGoalmouth` | `shotGoalmouth` | Optional 3x3 goalmouth placement view; currently hidden when validation is suspicious. |

## Zone Di Tiro

`Zone di tiro` is not a raw shot map. The payload does not expose raw shot events or exact shot coordinates. It exposes `HEATMAP_DATA.<playerId>.shotField`, a 12x8 aggregated grid.

The frontend aggregates the 12x8 cells into six ATT macro-zones:

- `Area piccola`
- `Centro area`
- `Lati area`
- `Limite area`
- `Fuori area centrale`
- `Corridoi larghi`

Visible zones are colored by total xG. Zone text shows shot count and xG per shot when readable. Full details remain available in SVG tooltips.

Cell contract:

```text
[shots, goals, xg_sum, xg_avg, avg_distance, shots_on_target_count, flag]
```

Tooltip contract:

- zone name;
- shots;
- goals;
- total xG;
- xG per shot;
- shots on target, when available.

## Orientation Caveat

The frontend can safely reason about:

- `low-y` lanes;
- `high-y` lanes;
- central lanes.

It cannot prove tactical left/right orientation from the current frontend contract alone. Labels should stay neutral unless the backend/export layer documents the coordinate orientation explicitly.

## Goalmouth Status

`shotGoalmouth` is treated as unvalidated.

The runtime hides the goalmouth panel when validation detects suspicious central collapse, such as one cell or the center column dominating the 3x3 grid. Do not present `shotGoalmouth` as reliable shot placement until backend/export validation confirms:

- true goalmouth coordinates;
- correct axis orientation;
- correct outcome filtering;
- missing coordinates preserved as missing rather than defaulted.

## QA Checklist For Future ATT Pages

- Confirm `shotField` is a 12x8 grid.
- Confirm `sum(shotField shots) == shotN`.
- Confirm `sum(shotField goals) == shotGoals`.
- Confirm `sum(shotField xG)` is consistent with `shotXg`.
- Inspect `low-y`, central, and `high-y` distributions without calling them left/right.
- Check hidden `Corridoi larghi` volume.
- Check both Hojlund-style and Lautaro-style stress cases.
- Confirm `Zone di tiro` renders on desktop.
- Confirm mobile width around 390px has no horizontal overflow.
- Confirm DEF/MID pages are unchanged.
- Confirm no goalmouth panel appears when `shotGoalmouth` is suspicious.
- Confirm no console errors.
